Adding contextualised help to a UI brick

The key concept in designing contextualised UI brick help is context. 

Think of a user in their particular context. Write for the most typical user of the brick in question. Keep it very short. You can add links to pertinent information in the other documentation types, how-tos, concepts, training, etc. This lets you stage the reading experience.

Although six article types are provided, the best strategy is to use one or two. This forces you to think of the type of content which is the most pertinent. If you end up wanting to use all six types, maybe you will need to talk to developers about where the documentation access is made available, or about the the ergonomics and clarity of the UI.
